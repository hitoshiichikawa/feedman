# Requirements Document

## Introduction

#120 で導入したフィード内検索では、検索を実行すると右ペイン全体が検索結果ビューに
差し替わり、フィード内検索バーを含むフィードヘッダ領域が画面から消える挙動になっている。
そのため、検索結果を見ながらキーワードを微調整して再検索したい場合に、いったん検索結果を
解除して通常の記事一覧に戻ってから検索バーに再入力する必要があり、検索→絞り込み→再検索の
反復が著しく阻害される。本仕様変更では、フィード内検索の結果表示中もフィードヘッダ領域に
フィード内検索バーを残置し、現在の検索キーワードが入力欄に反映された状態で再編集・再検索
できるようにする。横断検索（ヘッダー常設の横断検索バー）の挙動は本変更の対象外とし、#120
で確定済みの既存 AC（横断検索バーのヘッダー常設、検索結果表示、ユーザー隔離、既存記事操作
との整合、非機能要件）は本変更で書き換えない。

## Requirements

### Requirement 1: フィード内検索結果表示中のフィード内検索バー残置

**Objective:** As a Feedman ユーザー, I want フィード内検索の結果を見ている間もフィード内検索バーが画面上に残っていること, so that 結果を見ながら別ペインへ移動せずキーワードを編集して再検索できる

#### Acceptance Criteria

1. While ユーザーがフィード内検索の結果表示状態にある間, the 検索 UI shall 対象フィードのフィード内検索バーを画面上に表示し続ける
2. While フィード内検索の結果表示状態にある間, the 検索 UI shall 現在の検索キーワードをフィード内検索バーの入力欄に反映した状態で表示する
3. When ユーザーがフィード内検索の結果表示中にフィード内検索バーの入力欄を編集したとき, the 検索 UI shall ユーザー入力に追随して入力欄の表示値を更新する
4. When ユーザーがフィード内検索の結果表示中にフィード内検索バーで検索実行操作（送信）を行ったとき, the 検索 UI shall 編集後のキーワードを用いて当該フィードを対象とする新しい検索を開始する
5. When ユーザーがフィード内検索の結果表示中にフィード内検索バーの入力欄を空にして検索実行操作を行ったとき, the 検索 UI shall 新しい検索を開始せず直前の検索結果表示状態を維持する
6. When ユーザーがフィード内検索の結果表示中にフィード内検索バーのクリア操作を行ったとき, the 検索 UI shall 検索結果表示を解除し検索実行前の記事一覧表示状態へ戻す

### Requirement 2: 検索バー残置のスコープ限定

**Objective:** As a Feedman ユーザー, I want 残置されるのは現在の検索対象フィードに対応する検索バーだけであること, so that 横断検索結果や別フィードでの検索状態が混在して表示されない

#### Acceptance Criteria

1. While 横断検索の結果表示状態にある間, the 検索 UI shall フィード内検索バーをフィードヘッダ領域に表示しない
2. While フィード内検索の結果表示中にユーザーが左ペインで別のフィードを選択したとき, the 検索 UI shall 検索結果表示を解除し選択先フィードの通常の記事一覧表示状態へ遷移する
3. If ユーザーが特定のフィードを開いていない（フィード未選択）状態のとき, the 検索 UI shall フィード内検索バーを表示しない

### Requirement 3: 既存挙動との整合と非回帰

**Objective:** As a Feedman ユーザー, I want 本変更が #120 で確定した検索・記事操作・ユーザー隔離の挙動を変えないこと, so that 既存の検索機能・記事閲覧導線が回帰せずに動作し続ける

#### Acceptance Criteria

1. The 検索 UI shall #120 で導入したヘッダー横断検索バーの常設表示と横断検索の実行挙動を本変更によって変更しない
2. The 記事検索機能 shall 本変更によって検索対象範囲（横断検索 / フィード内検索の対象記事集合）およびユーザー隔離の判定基準を変更しない
3. The 検索結果画面 shall 本変更によってフィード内検索結果の表示順・空状態表示・ローディング表示・エラー表示・記事選択時の閲覧挙動を変更しない
4. When ユーザーが本変更導入後に通常の記事一覧表示（検索未実行）でフィードを開いたとき, the 検索 UI shall フィードヘッダ領域に従来と同じ要素群を表示する

## Non-Functional Requirements

### NFR 1: 既存挙動の非回帰

1. The 記事一覧画面 shall フィード内検索を一度も実行しない通常利用において、本仕様変更導入前と同じ記事一覧表示・選択挙動・既読／スター操作挙動を維持する
2. The 検索結果画面 shall フィード内検索結果の表示領域・記事カード構造・選択時の本文閲覧挙動を本仕様変更導入前と同一に保つ

### NFR 2: 再検索操作の即応性

1. While フィード内検索の結果表示中にユーザーがフィード内検索バーの入力欄を編集している間, the 検索 UI shall 入力に対する画面表示の更新がユーザーに即時として知覚できる程度の遅延（タイピング遅延としてユーザーが認識しない範囲）で完了する

## Out of Scope

- 横断検索（ヘッダー常設バー）の結果表示中のヘッダー検索バー挙動変更（#120 で常設済みのため変更不要）
- フィード内検索の結果表示中における「全て／未読／スター」フィルタタブの表示・有効化／無効化仕様（本 Issue の重点要件はフィード内検索バー残置に限定。フィルタタブの取り扱いは別 Issue で検討）
- 検索結果に対する事後絞り込み（期間・既読状態などのフィルタ追加）
- 検索履歴・サジェスト・最近のキーワード表示
- 検索キーワード入力時のリアルタイム検索（debounce 自動実行）。本仕様では送信操作によって新しい検索を開始する
- 横断検索・フィード内検索の検索方式・検索対象範囲・ユーザー隔離・記事操作整合の変更（#120 で確定済み）

## Open Questions

- フィード内検索の結果表示中に「全て／未読／スター」フィルタタブをフィードヘッダ領域に残すか、非表示・無効化・残置のいずれにするかは本 Issue の重点要件に含まれていない。残置する場合の挙動（検索結果に対する追加絞り込みとして動作させるか、無効化して見た目だけ残すか）にも依存するため、ユーザー価値判断を運用者に確認したい
- フィード内検索の結果表示中に手動更新ボタン（#115 で導入）をフィードヘッダ領域に残すかどうかは、フィードヘッダ領域に何を残すかという同じ判断点に属する。残す場合に検索結果との関係（更新後に再検索を発火するか）も含めて運用者に確認したい
