# Requirements Document

## Introduction

左ペインのフィード一覧では、登録フィードの名称がサイトを表すタイトル（例「Yahoo!ニュース」）ではなく
フィード URL のまま表示されており、複数フィード登録時に見分けが困難になっている。原因は、新規登録時に
初期タイトルとしてフィード URL が保存され、その後バックグラウンドフェッチでフィードを正常にパースして
正しいタイトルを取得しても、フェッチ状態を書き込む永続化処理にタイトル・サイト URL が含まれないため、
保存値が初期値（URL）のまま更新されないことにある。本要件は、フェッチ成功時にパース済みのタイトル・
サイト URL がフィードへ永続化され、結果として一覧・詳細取得 API のレスポンスおよび左ペイン表示が正しい
タイトルになることを、不具合修正のスコープに絞って定義する。

## Requirements

### Requirement 1: フェッチ成功時のフィードタイトル・サイト URL の永続化

**Objective:** As a フィードリーダー利用者, I want 登録フィードが最初のフェッチ・パースに成功したタイミングでサイトを表す正しいタイトルとサイト URL が保存されること, so that 左ペインのフィード一覧で URL ではなくサイト名でフィードを識別できる

#### Acceptance Criteria

1. When フィードのフェッチが成功しフィード内に定義されたタイトルがパースされたとき, the Feed Fetcher shall そのパース済みタイトルをフィードの保存タイトルとして反映する
2. When フィードのフェッチが成功しフィード内に定義されたサイト URL がパースされたとき, the Feed Fetcher shall そのパース済みサイト URL をフィードの保存サイト URL として反映する
3. When フェッチ成功時のフィード状態更新が完了したとき, the Feed Fetcher shall タイトル・サイト URL を含むパース結果をフィードの永続化先へ書き込む
4. The Feed Fetcher shall フェッチ成功時の状態更新において、これまで保存されていたフェッチ状態項目（フェッチ状態・連続エラー数・エラーメッセージ・次回フェッチ時刻・ETag・Last-Modified）も従来どおり同一の更新で反映する

### Requirement 2: パース結果が不完全な場合の既存タイトルの保護

**Objective:** As a フィードリーダー利用者, I want パース結果にタイトルが含まれない場合や境界的な入力のときに既存の表示名が壊されないこと, so that 一度正しく表示されたフィード名が予期せず空欄や不正な値に変わらない

#### Acceptance Criteria

1. If フェッチは成功したがパース済みタイトルが空であるとき, the Feed Fetcher shall フィードの保存タイトルを空値で上書きしない
2. If フェッチは成功したがパース済みサイト URL が空であるとき, the Feed Fetcher shall フィードの保存サイト URL を空値で上書きしない
3. While フィードの保存タイトルが初期値（フィード URL）のままであるとき, when パース済みタイトルが取得できたとき, the Feed Fetcher shall 初期値をパース済みタイトルへ置き換える

### Requirement 3: フェッチ失敗・未変更時のタイトル・サイト URL の維持

**Objective:** As a フィードリーダー利用者, I want フェッチに失敗したり内容が未変更だったときに既存のタイトル・サイト URL が維持されること, so that 一時的な取得失敗で表示名が劣化（URL へ戻る等）しない

#### Acceptance Criteria

1. If フィードのフェッチまたはパースが失敗し状態更新が行われるとき, the Feed Fetcher shall その更新によって既存の保存タイトル・サイト URL を破壊しない
2. When フィード内容が未変更（条件付き GET で未変更応答）でフェッチ状態のみ更新されるとき, the Feed Fetcher shall 既存の保存タイトル・サイト URL を維持する

### Requirement 4: 取得 API レスポンスおよび一覧表示への反映

**Objective:** As a フィードリーダー利用者, I want 一覧・詳細取得時にサイトを表すタイトルが返ること, so that 左ペインのフィード一覧が正しい名称で表示される

#### Acceptance Criteria

1. When フィードのタイトル・サイト URL が永続化された後に購読一覧取得が行われたとき, the Subscription API shall レスポンスのフィードタイトルとしてパース済みのサイトタイトルを返す
2. While フィードがフェッチ・パース成功済みであるとき, the Feed List View shall フィード一覧でフィード URL ではなく保存されたサイトタイトルを表示する

## Non-Functional Requirements

### NFR 1: 既存挙動の互換性

1. The Feed Fetcher shall 本修正導入後も、フェッチ成功・失敗・未変更の各経路における従来のフィード状態項目（フェッチ状態・連続エラー数・エラーメッセージ・次回フェッチ時刻・ETag・Last-Modified）の更新結果を変えない
2. The Subscription API shall 本修正導入後も、フィードタイトル以外の既存レスポンス項目（フィード URL・購読 ID・フェッチ間隔・未読件数等）の値と形式を変えない

### NFR 2: 検証可能性

1. The タイトル・サイト URL 永続化挙動 shall 正常系（パース成功で反映）・空入力（パース済みタイトル/サイト URL が空のとき非破壊）・異常系（フェッチ/パース失敗時に非破壊）の各ケースについて、再現可能な自動テストで検証できる形であること

## Out of Scope

- 取得 API のスキーマ変更・新規エンドポイント追加（既存のフィードタイトル項目をそのまま利用する）
- 左ペイン UI のレイアウト・表示ロジックの改修（保存タイトルが正しくなれば既存表示で解消される前提）
- フィードタイトルのユーザーによる手動編集・上書き機能
- フィードタイトル以外のメタデータ（favicon・説明文等）の取得・永続化仕様の変更
- 既に保存済みの誤ったタイトル（URL のまま）に対する一括バックフィル処理（次回フェッチ成功時に自然に更新される前提とし、本要件では強制移行を求めない）
