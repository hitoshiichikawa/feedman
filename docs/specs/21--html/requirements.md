# Requirements Document

## Introduction

Feedman の記事本文は現状、記事展開表示コンポーネント（`web/src/components/item-detail.tsx`）で
取得済み HTML をそのまま DOM へ挿入して描画している。バックエンドではフィード保存時・API 応答時に
HTML サニタイズ（許可リストベース）を実施済みだが、フロントエンド側には二重防御（多層防御）が無く、
バックエンドのサニタイズが想定外に弱まった場合・経路を迂回したコンテンツが届いた場合に XSS の
リスクが残る。本要件は、記事本文をフロントエンドでも描画前にサニタイズし、`<script>` やインライン
イベントハンドラなどの危険な要素・属性を除去しつつ、許可された装飾タグによる正常な記事本文の表示を
従来通り維持することを目的とする。バックエンド側のサニタイズ設定は変更せず、フロント側の防御層追加に
スコープを限定する。

## Requirements

### Requirement 1: 記事本文のクライアント側サニタイズ

**Objective:** As a Feedman 利用者, I want 記事本文が表示直前にフロントエンドでもサニタイズされること, so that バックエンド防御層を迂回・劣化した危険コンテンツが届いてもブラウザ上で実行されない

#### Acceptance Criteria

1. When 記事本文が記事展開表示に描画されるとき, the 記事展開表示 shall 記事本文 HTML を描画前にクライアント側でサニタイズした結果を DOM へ挿入する
2. The 記事展開表示 shall サニタイズ前の生の記事本文 HTML を DOM へ直接挿入しない
3. When 記事本文が空文字列のとき, the 記事展開表示 shall 何も描画せず空のコンテンツ領域を表示する
4. While 表示中の記事が同一であるとき, the 記事展開表示 shall 同一の記事本文に対して常に同一のサニタイズ結果を描画する

### Requirement 2: 危険な要素・属性の除去

**Objective:** As a Feedman 利用者, I want 危険なタグや属性が記事本文から取り除かれること, so that 悪意あるフィードによるスクリプト実行・情報漏洩から保護される

#### Acceptance Criteria

1. If 記事本文に `<script>` 要素が含まれるとき, the 記事展開表示 shall 当該要素を描画前に除去する
2. If 記事本文に `<iframe>` または `<style>` 要素が含まれるとき, the 記事展開表示 shall 当該要素を描画前に除去する
3. If 記事本文の要素に `onerror` などの `on*` インラインイベントハンドラ属性が含まれるとき, the 記事展開表示 shall 当該属性を描画前に除去する
4. If 記事本文の URL 属性に `javascript:` などの危険なスキームが含まれるとき, the 記事展開表示 shall 当該属性を無効化または除去する

### Requirement 3: 許可された記事本文の表示維持

**Objective:** As a Feedman 利用者, I want リンク・段落・強調などの許可された装飾が従来通り表示されること, so that サニタイズ追加後も記事の可読性が損なわれない

#### Acceptance Criteria

1. While 記事本文がバックエンドの許可タグ集合（段落・改行・リンク・リスト・引用・整形済み・コード・強調・画像）のみで構成されているとき, the 記事展開表示 shall 当該タグをサニタイズ後も保持して描画する
2. When 記事本文に許可された URL 属性を持つリンクまたは画像が含まれるとき, the 記事展開表示 shall 当該リンク・画像をサニタイズ後も保持して描画する
3. The 記事展開表示 shall フロント側サニタイズで保持する許可タグ・許可属性の集合をバックエンドのサニタイズポリシーと整合する範囲に維持する

## Non-Functional Requirements

### NFR 1: 多層防御の維持（セキュリティ）

1. The Feedman shall 本要件の対応によってバックエンド側のサニタイズ処理・許可リストポリシーを緩和または無効化しない
2. The 記事展開表示 shall バックエンドサニタイズの有無に関わらず、フロント側サニタイズを記事本文描画の必須経路として常に適用する

### NFR 2: 既存表示の互換性（互換性）

1. While 記事本文が既に許可タグのみで構成されているとき, the 記事展開表示 shall 本要件導入前と視覚的に等価な描画結果を表示する

## Out of Scope

- バックエンドのサニタイズ設定（`internal/security/content_sanitizer.go` の bluemonday ポリシー）の変更・緩和
- 記事本文以外の HTML 挿入箇所のサニタイズ（現状、生 HTML を DOM 挿入しているのは記事展開表示の記事本文のみ）
- 記事本文の整形・スタイリング（CSS による装飾）の見直し
- バックエンドとフロントの許可タグ集合の差分を統一する追加リファクタリング（整合の「範囲維持」は要件に含むが、ポリシー単一化は対象外）
