# Requirements Document

## Introduction

現状、はてなブックマーク数とお気に入り（スター⭐️）は記事を展開して詳細を開かないと
閲覧・操作できない。これにより、一覧画面のままでは「はてブ数の把握」「スターの ON/OFF」が
行えず、利用者は記事を 1 つずつ展開する必要があり効率が悪い。本要件では、はてブ数と
スター⭐️を 3 種類すべての記事一覧（通常記事一覧 / スター横断一覧 / 検索結果一覧）の
各行右端に常設し、その場でトグル可能にする。同時に、責務集約のため詳細ヘッダーから
両表示を撤去する。検索結果一覧のはてブ未取得表示の扱いは、検索 API レスポンスに
はてブ取得日時を追加して 3 一覧の挙動を統一する方針（Issue コメントで Option C を採用）
を採る。

## Requirements

### Requirement 1: 一覧行右端メタ表示（はてブ数 + スター⭐️）

**Objective:** As a 記事一覧を閲覧する利用者, I want 各記事行の右端で「はてブ数」と「スター⭐️」を常時確認できる, so that 記事を展開せずに重要な指標と状態を一目で把握できる

#### Acceptance Criteria

1. The 記事一覧行 shall タイトル行の右端に「はてブ数（アイコン + 数値）」と「スター⭐️トグル」を同一行内に表示する
2. The 記事一覧行 shall 「はてブ数」を「スター⭐️トグル」より左側に配置する
3. When 当該記事のはてブ取得日時が未取得（値なし）であるとき, the 記事一覧行 shall はてブ数値の代わりに `-` を表示する
4. When 当該記事のはてブ取得日時が取得済み（値あり）であるとき, the 記事一覧行 shall 当該記事のはてブ数（整数値）をそのまま表示する
5. When 当該記事がスター付きであるとき, the スター⭐️トグル shall 塗りつぶし（黄色）状態のアイコンを表示する
6. When 当該記事がスター無しであるとき, the スター⭐️トグル shall アウトライン（未スター）状態のアイコンを表示し、クリック可能な状態を保つ
7. The 記事一覧行 shall 既存の「公開日時」「(推定) バッジ」「概要」表示を維持し、右端メタ表示の追加により撤去しない

### Requirement 2: 一覧上スター⭐️トグル操作

**Objective:** As a 記事一覧を閲覧する利用者, I want 記事を展開せずに一覧行上でスターの ON/OFF を切り替えられる, so that 興味を持った記事を即座にお気に入り登録できる

#### Acceptance Criteria

1. When 利用者が一覧行のスター⭐️トグルをクリックするとき, the 一覧行 shall 当該記事のスター状態を反転させる更新要求をバックエンドへ送信する
2. When 利用者が一覧行のスター⭐️トグルをクリックするとき, the 一覧行 shall バックエンド応答を待たずに UI 表示を新しいスター状態へ即時反映する（楽観更新）
3. When 利用者が一覧行のスター⭐️トグルをクリックするとき, the 一覧行 shall 同じクリックイベントが行クリック（記事展開）として伝播することを防止する
4. If スター切替要求が失敗するとき, the 一覧行 shall 楽観更新を取り消し、元のスター状態へ UI を戻す
5. When スター切替が成功し UI に反映された後, the 同記事を含む他の一覧 shall 一貫したスター状態を表示する（通常一覧 / スター横断一覧 / 検索結果一覧の間で食い違わない）

### Requirement 3: 詳細ヘッダーからのメタ撤去

**Objective:** As a 記事を展開して閲覧する利用者, I want 詳細ヘッダーに表示重複が無くシンプルである, so that 一覧側との責務分離が明確で詳細領域は本文閲覧に集中できる

#### Acceptance Criteria

1. The 記事詳細ヘッダー shall はてなブックマーク数（アイコン + 数値）表示を表示しない
2. The 記事詳細ヘッダー shall スター⭐️切替トグルを表示しない
3. The 記事詳細ヘッダー shall タイトル、著者、元記事リンク、本文の各既存要素を維持する
4. The 記事詳細 shall 展開時に未読記事を自動的に既読化する既存挙動を維持する

### Requirement 4: 3 一覧（通常 / スター横断 / 検索結果）への適用統一

**Objective:** As a 利用者, I want 通常記事一覧・スター横断一覧・検索結果一覧のいずれにおいても右端メタ表示と操作が同一である, so that 画面間で挙動の差異を学習し直す必要がない

#### Acceptance Criteria

1. The 通常記事一覧 shall Requirement 1 と Requirement 2 の挙動を満たす
2. The スター横断一覧 shall Requirement 1 と Requirement 2 の挙動を満たす
3. The 検索結果一覧 shall Requirement 1 と Requirement 2 の挙動を満たす
4. The 検索結果一覧 shall 既存のフィード識別バッジ（フィードタイトル + favicon）と公開日時の表示を維持する
5. The スター横断一覧 shall 既存のフィードタイトル行内表示を維持する

### Requirement 5: 検索 API 応答へのはてブ取得日時追加（Option C）

**Objective:** As a 検索結果一覧を閲覧する利用者, I want 検索結果でも「はてブ数 0 件」と「未取得」を区別して表示する, so that 通常一覧・スター横断一覧と一貫した情報を受け取れる

#### Acceptance Criteria

1. The 検索結果取得 API shall 応答の各記事項目に「はてブ取得日時」（日時文字列、または未取得を示す値なし） を含める
2. The 検索結果取得 API shall 既存の応答フィールド（記事 ID / フィード ID / タイトル / リンク / 概要 / 公開日時 / 公開日時推定フラグ / はてブ数 / フィードタイトル / favicon / 既読状態 / スター状態） を保持する
3. When 検索結果の記事のはてブ取得日時が未取得（値なし）であるとき, the 検索結果一覧行 shall はてブ数値の代わりに `-` を表示する
4. When 検索結果の記事のはてブ取得日時が取得済み（値あり）であるとき, the 検索結果一覧行 shall 当該記事のはてブ数（整数値）をそのまま表示する

### Requirement 6: 既存挙動の非回帰

**Objective:** As a 既存利用者, I want 既存の一覧挙動が壊れない, so that 安心して新機能を受け入れられる

#### Acceptance Criteria

1. The 記事一覧 shall 既読記事の不透明度低下（薄表示）を維持する
2. The 記事一覧 shall 一覧末尾までスクロールした際の追加ページ自動読み込み（無限スクロール）挙動を維持する
3. The 通常記事一覧 shall フィルタタブ（すべて / 未読 / スター付き） の切替挙動を維持する
4. The 記事一覧行 shall タイトルリンク（外部記事リンク）のクリックが行展開イベントへ伝播しない既存挙動を維持する
5. The 記事一覧行 shall 公開日時の (推定) バッジ表示を維持する
6. The スター横断一覧 shall コンテキストタイトル「お気に入り」のヘッダ表示を維持する
7. The 検索結果一覧 shall 横断検索時のフィード識別バッジ表示と空状態 / エラー状態 / ローディング状態の出し分けを維持する

## Non-Functional Requirements

### NFR 1: アクセシビリティ

1. The 一覧上スター⭐️トグル shall 現在のスター状態（「スターを付ける」/「スターを解除する」）を支援技術が読み上げ可能な形式で提示する
2. The 一覧上スター⭐️トグル shall 現在の押下状態（押下中 / 未押下） を支援技術が認識可能な形式で提示する
3. The 一覧上スター⭐️トグル shall 最小 32px × 32px のクリック可能ヒット領域を確保する
4. The 一覧上スター⭐️トグル shall キーボード操作（Tab フォーカス、Enter または Space キーでの発火）でスター切替を実行可能にする

### NFR 2: イベント独立性と応答性

1. When 利用者が一覧上スター⭐️トグルをクリックするとき, the 一覧行 shall 行クリック（記事展開）を発火させない
2. The 一覧上スター⭐️トグル shall クリックから 100ms 以内に UI 上の状態反転を表示する（楽観更新）
3. If スター切替要求が失敗するとき, the 一覧上スター⭐️トグル shall 失敗確定後 200ms 以内に元の状態へ UI を戻す

### NFR 3: 後方互換 / 非回帰

1. The 既存の記事一覧応答スキーマ（通常一覧 / スター横断一覧） shall 既存フィールドを維持し、本変更で削除しない
2. The 一覧側スター⭐️トグル shall 一覧上のスター状態を読み取れる既存の自動テスト用識別子（記事ごとのスターアイコン識別子と記事行識別子） を維持または等価な後継識別子で置き換える
3. The 詳細ヘッダー側で撤去される自動テスト用識別子 shall 一覧側で新規付与される識別子と衝突しない命名で再構成される

## Out of Scope

- はてブ数による並び替え・フィルタ機能
- はてなスター等、はてブ以外の指標表示
- 複数選択して一括スターする一括操作 UI
- 一覧 / 詳細レイアウトの全面刷新
- バックエンド側ではてブ取得頻度・取得対象の変更
- 一覧行からのコメント閲覧・外部はてブページへの遷移

## Open Questions

- なし（検索 API へのはてブ取得日時追加方針は Issue コメントで Option C 採用と確定済み）
