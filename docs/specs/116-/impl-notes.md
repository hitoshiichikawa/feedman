# Implementation Notes - Issue #116

## 概要

ItemDetail コンポーネントのヘッダーを再構成し、はてブ数とスター切替アイコンを
タイトル行右側に集約。旧アクションバーを廃止し、「元記事を開く」リンクを著者名行へ
中点区切りで再配置した。既存挙動（本文サニタイズ、300px 折りたたみ、続きを読む、
自動既読化、楽観的 UI 更新、テスト識別子）は維持した。

## 既存のスター切替テスト識別子・aria 周りの方針判断

- `data-testid="star-toggle"` は既存テスト（`item-list.test.tsx`、本ファイル）が依存しているため
  完全に温存した（Req 5.4）
- 旧実装は文字列マッチング（"スター" / "スター解除"）に依存していたが、新実装は icon-only の
  ため、テストはアクセシブル名（`aria-label`）と `aria-pressed` ベースに移行
- `aria-label` の文言は「スターを付ける」（未スター時）/「スターを解除する」（スター付き時）と
  動詞型に統一した。これは AC で具体的な文字列が指定されていなかったため、操作の用途と
  状態を支援技術へ正確に伝える文言を選んだ
- `aria-pressed` は `true` / `false` 文字列で出力される（React の boolean 属性 → ARIA bool 変換）。
  テストでは文字列 `"true"` / `"false"` で比較
- `title` 属性も同じ文言で付与し、マウスホバー時のツールチップとしても機能させた
